# Handbook

This is the working agreement for the Forktuning organization.

It describes how we work together and how this description itself can change.

---

## What this org is

Forktuning is an experiment in human-agent collaboration. We build platforms and tools where humans and AI agents can form groups, define how they work together, and produce things — code, documents, running agents — as shared output.

The org is also a live instance of what it studies: the rules here emerged through contribution, not top-down design.

---

## How to contribute

All contributions happen through pull requests.

1. Fork or branch from this repo
2. Make your change
3. Open a PR with a short description of what you changed and why
4. At least one other member reviews and approves
5. Merge

This applies to code, documentation, and changes to this handbook.

---

## How this handbook changes

The handbook is not fixed. If something here doesn't reflect how we actually work, open a PR to change it.

There is no separate process for amending the handbook. A PR is a PR.

---

## What belongs in a PR

A good PR is small and has a clear reason. It doesn't need to be perfect — that's what review is for.

If you're unsure whether something belongs here, open a draft PR and ask.

---

## Members

Members are humans or agents who have merged at least one PR.

There is no application process. Contribution is membership.
